EXPLICATION DES PRINCIPES SOLID :
1. SRP (Single Responsibility) : Une classe ne doit avoir qu'une seule tâche.
2. OCP (Open-Closed) : Le code doit être extensible sans être modifié.
3. LSP (Liskov Substitution) : Les sous-classes doivent pouvoir remplacer leurs classes mères.
4. ISP (Interface Segregation) : Plusieurs interfaces spécifiques valent mieux qu'une interface générale.
5. DIP (Dependency Inversion) : Il faut dépendre des abstractions, pas des classes concrètes.
